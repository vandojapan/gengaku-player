## Implementation Order

1. Verify already-implemented commands against the target game: `3007`, `3008`, `3015`, `3020`, `3026`.
2. Produce reference logs for unresolved observed subfeatures: `3018` operation 1, `3021` subcommand 3, `3025`, `3032`.
3. Implement missing dispatch only after reference behavior is known: `3025`, `3032`, and any missing sub-behavior for `3018`/`3021`.
4. Add regression tests around the exact observed parameter shapes and a full target-flow screenshot/PNG comparison.

## Non-Negotiables

- Do not replace any command with a stub/no-op.
- Do not treat a command as complete just because dispatch exists; observed parameter variants must be covered.
- If a command affects timing, framebuffer, file output, or persistent state, compare against Maniacs/RPG_RT before implementation.
